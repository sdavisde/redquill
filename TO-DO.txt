## Part B — From-scratch setup for a new developer

1. Install Rust stable (rustup.rs), then clone the repo.
2. `cargo build` — no external services, no env vars, single static binary.
3. `cargo test` — unit + integration (integration tests create throwaway git
   repos in tempdirs; git must be on PATH).
4. `cargo run` — opens the TUI on the current repo's working-tree diff (spec 04):
   scroll with `j`/`k` and `Ctrl-d`/`Ctrl-u` (arrows/PgUp/PgDn alias), quit with
   `q`. Needs a real terminal; in a TTY-less environment it exits cleanly with a
   terminal I/O error. `--staged` and ref ranges select other diff targets.
5. Quality bar before any commit: `cargo build && cargo test && cargo clippy -- -D warnings && cargo fmt --check`.
