## `main.rs::run()` wiring (4.2)

Replaced the Task-2 per-file `println!` loop (and its `patch_letter` helper)
with:

```rust
let files = diff::parse_patches(&patches);   // FR-diff-wire-1
let mut summary = diff::summarize(&files);

if matches!(target, DiffTarget::WorkingTree) {
    let untracked = runner
        .status()?
        .into_iter()
        .filter(|entry| entry.kind == ChangeKind::Untracked)
        .count();
    summary.files += untracked;
}

println!(
    "{} files, {} hunks, +{} -{}",
    summary.files, summary.hunks, summary.added, summary.removed
);
```

`run()` still returns `anyhow::Result<()>` and contains no `unwrap()` /
`expect()`.

**Deviation/design choice — untracked-file handling:** `git diff` never
surfaces untracked files, so the working-tree summary would otherwise
silently omit them. Untracked files have no diff body (no hunks, no
added/removed lines), so rather than reintroducing a separate per-file `?`
listing (which is exactly the Task-2 shape this task removes), their count is
folded into `summary.files`. This keeps the file count complete and honest
while preserving the single-summary-line contract. Documented per the task's
explicit instruction ("Preserve the working-tree untracked-file handling
only if it still makes sense for a summary (else fold untracked count into
files); document the choice").
