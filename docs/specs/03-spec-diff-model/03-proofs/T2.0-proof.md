## Algorithm summary

Tokenize each line's chars into maximal runs: word runs (alnum + `_`) and
whitespace runs merge across any chars of their class; a punctuation run
only merges while the *same* char repeats, so `foo.bar()` tokenizes as
`foo`, `.`, `bar`, `(`, `)` (spec §9 default). An LCS DP over the token
sequences finds the longest common subsequence of *matched* tokens; every
unmatched token's char range becomes a changed span, with adjacent
unmatched tokens merged into one contiguous span. Span units are **char**
offsets into `Line.content` (spec §9). `word_diff_spans` is the only
function that runs this algorithm; `attach_word_spans` is the only caller,
positionally pairing removed/added lines within each contiguous
(non-`Context`) change run and leaving excess/unpaired/context/identical
lines with empty `changed_spans`.

A `MAX_LINE_TOKENS` guard (400 tokens/side) degrades pathologically long
lines to a single whole-line span instead of building an O(n·m) table, to
protect the 5k-line-diff instant-feel performance target.
