## Semantics implemented (documented for downstream `ui/` consumers, Tasks 4-5)

- `next_hunk`/`prev_hunk` cross file boundaries and skip zero-hunk files
  entirely (both when leaving the current file and when scanning through
  intermediate zero-hunk files); `None` at the model ends.
- `next_file`/`prev_file` always land at `{hunk: 0, line: 0}` of the adjacent
  file, including zero-hunk files (their header is a valid position); `None`
  at the model ends.
- All landing positions from `next_hunk`/`prev_hunk` reset `line` to `0`
  (position at the start of the landed-on hunk); `prev_hunk` crossing a file
  boundary lands on the *last* hunk index of the previous file (not hunk 0),
  since that is the hunk immediately preceding the current position.
