## Implementation summary (3.2–3.5)

- **3.2 `restore_terminal()`:** `disable_raw_mode()` +
  `execute!(stdout(), LeaveAlternateScreen, cursor::Show)`; both calls'
  errors are discarded (`let _ = ...`) so the function is idempotent and
  never panics whether or not a TTY/alt-screen/raw-mode state exists.
  Tagged `// FR-render-term-1 / FR-render-term-3` as the single shared
  teardown path for both normal exit (`Drop`) and the panic hook.
- **3.3 `TerminalGuard::enter()`:** enables raw mode, enters the alternate
  screen, builds the `Terminal<CrosstermBackend<Stdout>>`; each fallible
  step after raw mode is enabled calls `restore_terminal()` before
  returning `Err(...)` via `UiError::Io` (no `unwrap`/`expect`). The panic
  hook is installed via `PANIC_HOOK_INSTALLED: std::sync::Once` —
  `take_hook()` → `set_hook` wrapping a closure that calls
  `restore_terminal()` THEN invokes the previous (chained) hook, so the
  default panic message prints on an already-restored screen
  (FR-render-term-2). `Once` guarantees a single install across repeated /
  nested `enter()` calls, so hooks never stack.
- **3.4 `impl Drop for TerminalGuard`:** unchanged from T1.0's stub —
  already called `restore_terminal()`; tagged `// FR-render-term-3` and
  left as the sole teardown call site (no logic duplicated inline).
  `terminal_mut()` unchanged (already correct in the T1.0 stub).
- **3.5 traceability comments:** `// FR-render-term-1`, `// FR-render-term-2`,
  `// FR-render-term-3` placed at `enter()`'s raw-mode/alt-screen call,
  `enter()`'s panic-hook install, `restore_terminal()`'s doc line, and
  `Drop::drop`, respectively.
